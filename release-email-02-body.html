UX Enhancements

We have created an organized view of the elastic agent profiles. Now, you can view them by the type of elastic agent plugin (Docker Swarm, OpenStack, ECS etc.), making it easier to find a particular profile.


Administrators can now choose to edit a Pipeline with one-click from the Value Stream Map.


GoCD and Docker

GoCD now sends logs to docker by writing them to STDOUT. Users can access the logs anytime in both server and agent containers. The GoCD Agent docker image for the latest version of Alpine (3.6) is now available to use.


We have also made other improvements and bug fixes this release. A comprehensive list of changes can be found here.

Bux Fixes

We've also made several bug fixes including removing memory leaks from agent bootstrapper.

  • There were some file system resource and memory leaks that have been identified which affect versions of go-agent bootstrapper between 17.4 and 17.8. We recommend that users either upgrade their gocd server and agent versions to the latest version, or restart the agent process to release any memory and filesystem resources that may be locked by the agent process.

Breaking Change

If you are using the commercial Amazon ECS Elastic Agent Plugin, upgrading to GoCD 17.9.0 would require a plugin upgrade to V2.0.0.

Deprecations

These are a list of features that have been deprecated on GoCD along with the timeframe of their removal.

  • As of release 17.9, version 2 of the plugin info API is deprecated. This version of the API will be removed in 17.12.0 release (scheduled to be released in December 2017). Version 3 of the API is available, and users are encouraged to use it.
  • Going forward, all the existing functionality provided by the Authentication Plugin endpoints will be handled by the new Authorization Plugin endpoints. Plugin developers are encouraged to migrate their plugins to use the new Authorization Plugins. Support for the endpoint will be discontinued from version 18.1 (scheduled to be release in January 2018).
  • As of 17.5, support for in-built password file and LDAP/AD authentication have been deprecated. Support for these authentication mechanisms would be removed in 17.10 (scheduled to be in October 2017). Existing configuration will be automatically migrated to use the new in-built password file and LDAP/AD authentication plugins. No other actions necessary.
