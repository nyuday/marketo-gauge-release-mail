Performance Fixes and Improvements

This release fixes the performance lag that occurred when a user saved any configuration in GoCD. We have also optimized the application to significantly reduce the number of SQL calls executed.


We've also made other fixes and improvements in GoCD 18.4. A more comprehensive list of changes can be found in our release notes.

Updates to bundled Plugins

We have upgraded the bundled LDAP authentication plugin

and the File-based authentication plugin to version 1.0.2.


Deprecations

  • As of release 18.2 the following unversioned APIs have been deprecated and will be removed in June 2018 - Pipeline pause, unpause, unlock and schedule APIs . These APIs have been replaced with a versioned API and users are encouraged to use these instead.
  • The pipeline config API version 3 and version 4 have been deprecated and will be removed in June 2018. Version 5 of the pipeline config API has been introduced to add the new pipeline unlock behavior attribute.
  • The Elastic Agent API version 1 (and a development verion 2) has been deprecated. This will be removed in June 2018. Plugin developers should use version 3 of the Elastic Agent plugin API to allow adding support for agent status report and plugin status report. The Docker, Docker Swarm, and ECS plugins have been migrated to the new versions, and users should upgrade their plugins to the latest version to see these new features.
  • The Plugin Info API v3 has been deprecated as part of 18.3. This version of the API will be removed in July 2018. Version 4 of the API is available, and users are encouraged to use it.

Breaking Changes

The Agent Docker Image for Debian 7 will not be built with the newer version of GoCD. Users who want to build a GoCD agent image for debian 7 can fork this repository.


Known Issues

There are a few known minor issues when triggering pipelines on the new dashboard. You can view them here and if you're looking to contribute to GoCD, fixing these are a great way to get started :) !
