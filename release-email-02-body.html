UX Enhancements

We have made it easier to find the Value Stream Map (VSM) for your pipeline. Pipeline instances now have an explict link to VSM across the application for better discoverability. Also, pipeline run durations are now visible in the VSM.



OTHER CHANGES

We've made a few improvements to GoCD 17.11 like adding support for bitbucket & gitlab webhooks and support for parameters & template references in config-repo pipelines (pipeline as code).


We've made a few changes to the GoCD Server and the GoCD Agent Docker images.


We have also made several bug fixes, API and performance enhancements. A more comprehensive list of changes can be found in our release notes.

BREAKING CHANGES

  • We have moved over to logback from log4j for logging. GoCD will no longer honour log4j.properties file. Users managing log4j.properties file externally would have to update their scripts to use the logback.xml instead.
  • Support for phusion based docker server and agent images have been removed this release. We provide support for the docker images for GoCD agents across the following operating systems - Alpine 3.5, CentOS 6 & 7 , Debian 7 & 8, Ubuntu 12.04, 14.04 & 16.04.

Deprecations

  • With release 17.5.0 support for in-built password file and LDAP/AD authentication had been deprecated. In this release support for this has been removed in favour of the bundled LDAP and Password File plugins respectively.
  • As of release 17.5, authentication plugin endpoints have been deprecated. Support for these plugins will be removed in 18.1 (scheduled for January 2018). Plugin developers are encouraged to migrate their plugins to use the Authorization Plugins that were introduced.

For a complete list of features that have been deprecated on GoCD along with the timeframe of their removal, see our release notes.
